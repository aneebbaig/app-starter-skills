# No AI attribution, and nothing that reads as AI-written

This is a hard rule for every app these skills scaffold or retrofit. The repo
must look like a human developer wrote it. Apply this to the OUTPUT project,
not to this skills repo itself (a Claude Code plugin is openly a Claude Code
plugin).

## Never put AI references in the output repo

Do not add, commit, or push any reference to Claude, Anthropic, an AI, an
assistant, an agent, Copilot, ChatGPT, Cursor, or any AI tool anywhere in the
generated project:

- not in code or code comments
- not in README, docs, or any markdown
- not in commit messages, commit author name, or commit email
- not in package.json, pubspec.yaml, pyproject, config, or HTML meta/generator
- not as a co-author trailer or a "generated by" line

Never commit AI-tool files into the output repo: `CLAUDE.md`, `AGENTS.md`,
`.claude/`, `.cursor/`, `.github/copilot-instructions.md`, `.aider*`,
`.continue/`, or similar. Add them to `.gitignore`. If a CLI or framework
injects one, delete it and confirm it stays ignored.

The only acceptable use of "AI" in an output repo is describing a product
feature (for example "AI-graded practice with your own API key"). That describes
a capability. It must never imply the app itself was built by an AI.

## Nothing may READ as AI-written

- No em dashes or en dashes anywhere. Use commas, hyphens, or separate
  sentences. Same for curly quotes, curly apostrophes, and the ellipsis
  character. Use plain ASCII: `'`, `"`, `...`.
- No AI-cadence prose. Avoid the tells: over-balanced "not only X but also Y",
  "it's worth noting", "in the world of", "delve", "seamless", "robust",
  "leverage", relentless tricolons. Write plain and direct.
- No AI-sounding or AI-referencing comments. Comments explain the code like a
  human dev would.

## Commits

Author commits as the human owner in normal conventional-commit style (`feat:`,
`fix:`, `chore:`, `docs:`). No AI co-author trailers. No "generated by" lines.

Before any commit, scan the diff for em dashes, curly punctuation, AI-cadence
text, and any AI reference. If in doubt, rewrite plainer.

## The mechanical scan (run the commands, do not eyeball)

Eyeballing misses things. Run these from the repo root and show the output.
All three must come back empty before any push to a public repo, and the
first two before any commit anywhere:

```bash
# 1. AI references in the working tree (tune the exclude list to the stack)
grep -rniE 'claude|anthropic|copilot|chatgpt|cursor|co-authored-by|generated (by|with)' . \
  --exclude-dir={.git,node_modules,.next,build,dist,.dart_tool,.venv} \
  --exclude='*.lock' --exclude='*-lock.*'

# 2. Smart punctuation in the working tree (em/en dash, curly quotes, ellipsis)
# perl, not grep -P: BSD grep on macOS has no -P and exits "clean" without scanning
git ls-files -z | xargs -0 perl -CSD -Mutf8 -ne \
  'print "$ARGV:$.: $_" if /[\x{2013}\x{2014}\x{2018}\x{2019}\x{201C}\x{201D}\x{2026}]/; close ARGV if eof'

# 3. Public repos only: the same terms across ALL history, plus author check
git log --all -i -E --grep='claude|anthropic|copilot|chatgpt|co-authored-by' --oneline
git log --all --format='%an %ae' | sort -u   # every author must be the human owner
git grep -i -E 'claude|anthropic|copilot|chatgpt' $(git rev-list --all) || true
```

A hit in the tree gets fixed before commit. A hit in history of a repo about
to go public means scrub history (filter-repo) or start a clean history; a
public repo is permanent once cloned or indexed, so treat the first push as
final. Also wire the CI leak guard from `git-and-ci.md` so a future slip
fails loudly.

## Open-source: the hard gate

For a public repo the bar is absolute. Never push anything that even hints an
AI was involved in building the project. Before the first public push, and
before every push after:

- Run the mechanical scan above; all clean.
- Scan the full working tree for AI-tool files (`CLAUDE.md`, `AGENTS.md`,
  `.cursor/`, `.claude/`, `.github/copilot-*`, `.aider*`, `.continue/`) and
  remove them. Confirm they are gitignored.
- Check commit author and email on every commit. All human. No bot or AI author.
- Check package metadata, HTML meta and generator tags, and CI config for
  generator or tool references.
- Read the README and docs out loud in your head: if any sentence sounds like
  a model wrote it, rewrite it plainer.

When unsure whether something reads as AI-written, it does. Rewrite it.
